## 1. 오전 삽질 요약

### 삽질 ①: 코드 버그 — v3 테스트가 실제론 v1 모델을 돌림

`analyzer.py`의 `_MODEL_DIR` 경로 생성 코드에 버그가 있었다.

```python
# 버그 있는 코드
_MODEL_DIR = Path(__file__).parent / f"models/koelectra-finetuned{'-v2' if DATA_VERSION == 'v2' else ''}/best"
# DATA_VERSION == 'v3' 이면 → models/koelectra-finetuned/best  (v1 원본 모델!)

# 수정 후
_MODEL_DIR = Path(__file__).parent / f"models/koelectra-finetuned-{DATA_VERSION}/best"
```

이 버그로 인해 v3로 설정하고 돌린 mixed_test 결과(F1 29%, FP 424건)는
**v3 모델이 아닌 v1 원본 모델**의 결과였다. v3 모델 자체의 성능은 측정조차 안 된 상태였다.

> 참고: 당시 사용한 테스트셋(mixed_test_01/02)은 정상 샘플로 K쇼핑 콜센터 데이터를 사용했고,
> 피싱 샘플은 train_ml.json에서 가져왔다는 별도의 문제도 있었다(삽질 ② 참고).

---

### 삽질 ②: 데이터 오염 — v3가 100% F1을 기록한 이유

코드 버그 수정 후 v3로 다시 mixed_test를 돌렸더니 **F1 100%, FP 0, FN 0**이 나왔다.

이는 v3 모델이 뛰어난 게 아니라 **답안지를 미리 외운 것**이었다.

**오염 구조:**

| | v3 학습 데이터 | mixed_test 정상 샘플 |
|---|---|---|
| 출처 | `normal_callcenter.json` (K쇼핑 업무처리 2,500개) | K쇼핑 콜센터 1,350개 |
| 관계 | 원본 데이터셋의 일부를 샘플링해서 학습 | 같은 원본 데이터셋에서 추출 |

v3 학습 시 K쇼핑 콜센터 데이터를 **전체 투입**했는데, mixed_test의 정상 샘플도 같은 K쇼핑 데이터였다.
콜센터 데이터에서 10~20%를 test용으로 미리 분리했어야 하는데, 전부 학습에 써버린 것이 원인이다.

## 2. 올바른 평가 — mixed_test_03

> v2와 v3가 동일한 결과: mixed_test_03의 정상 샘플(금융보험/다산콜센터/질병관리본부)은
> 두 모델 모두 처음 보는 도메인이므로 성능 차이가 나타나지 않았다.

---

### FN 분석 (미탐 16건 — 키워드 필터에서 걸러지지 않은 피싱)

| score | 내용 |
|---|---|
| 30 | 검찰 수사관 사칭 통화 전문 (지검/수사과 키워드 없이 우회) |
| 30 | "OO은행입니다. CVC 번호와 유효기간 말씀해 주실 수 있으신가요?" |
| 30 | "FIU 규정을 피하려면 800만원씩 두 번 출금하셔야 해요." |
| 15 | 대출 사기 유형 (한국자산관리공사 사칭, 장문) |
| 15 | "제3자에게 본인 명의로 빌려줬다던가 신분증 분실한 적 있으신가요?" |

**공통 패턴:** 키워드 점수가 15~30점으로 `ELECTRA_THRESHOLD`(현재 0, 운용 시 21) 미만이거나
모델이 문맥을 단일 발화로만 판단해 피싱 패턴을 놓침.

### FP 분석 (오탐 1건)

- "입금계좌를 기준으로 입금건당 100만원 이상은 금액에 대하여 일정 시간 인출 및 이체를 제한하는 제도입니다."
- 금융 제도 안내 문장인데 `금전요구` 카테고리로 오탐. 이체/인출 관련 표현이 피싱과 표면 유사.

## 3. 최종 모델 비교 정리

| | mixed_test_01/02 (오염) | mixed_test_03 (신규, 오염 없음) |
|---|---|---|
| **v1** (버그로 잘못 측정) | F1 29%, FP 424 | 미측정 |
| **v2** | F1 93~96%, FP 7 | **F1 82.8%, FP 1** |
| **v3** | F1 100% (오염) | **F1 82.8%, FP 1** |

- mixed_test_03 기준으로 v2와 v3 간 실전 성능 차이 없음
- v2의 mixed_test_01/02 결과(96%)와 mixed_test_03(83%) 간 갭은 테스트셋 정상 도메인 차이에서 기인
  (01/02는 K쇼핑 위주 → 학습 분포와 유사 / 03은 금융/행정/보건 → 새 도메인)
- Recall 71.9%는 키워드 필터(`ELECTRA_THRESHOLD=0`)를 정상 운용값(21점)으로 복원하면 달라질 수 있음
