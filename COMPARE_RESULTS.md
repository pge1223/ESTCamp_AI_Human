# 보이스피싱 탐지 모델 단계별 성능 비교

**데이터셋**: `test_ml.json` (57개 — 피싱 41개 / 정상 16개)  
**라벨 분포**: 기관사칭 19개 / 금전요구 17개 / 개인정보 17개  
**멀티라벨**: 기관사칭 / 금전요구 / 개인정보 독립 판단 (중복 가능)

---

## 결과 요약

| 단계 | Accuracy | Precision | Recall | F1 (피싱/정상) | F1 (macro) |
|------|:--------:|:---------:|:------:|:--------------:|:----------:|
| ① 키워드 규칙 기반 | 93% | 97% | 93% | 95% | 85% |
| ② KoELECTRA 베이스 (파인튜닝 전) | 72% | 72% | 100% | 84% | 27% |
| ③ KoELECTRA 파인튜닝 (현재 모델) | **100%** | **100%** | **100%** | **100%** | **95%** |

### 파인튜닝 모델 카테고리별 F1

| 기관사칭 | 금전요구 | 개인정보 |
|:-------:|:-------:|:-------:|
| 93% | 95% | 97% |

### 핵심 해석

- **① → ③**: 파인튜닝 모델이 키워드 규칙 대비 F1(bin) +5%p, F1(macro) +10%p 향상.  
  키워드 없이 문맥만으로 보이스피싱을 판단하는 능력이 추가됨.
- **② → ③**: 동일한 KoELECTRA 아키텍처라도 파인튜닝 없이는 F1(macro) 27%에 불과.  
  사전학습 언어모델 단독으로는 보이스피싱 판단이 불가능하며, 도메인 특화 학습이 필수임을 입증.
- **①이 ②보다 높은 이유**: 베이스 모델의 분류 헤드가 랜덤 초기화 상태라 대부분의 입력을 피싱으로 예측하는 편향 발생 (Recall=100%지만 정상 오탐 多). 키워드는 사람이 설계한 규칙이므로 Precision 97%로 높게 유지됨.

## 단계별 설명

### ① 키워드 규칙 기반 (슬라이딩 윈도우, AI 없음)

**방식**: 텍스트에서 사전 정의 키워드 직접 매칭. AI 추론 없음.

```
기관사칭 키워드: 검찰, 검사, 수사관, 경찰, 금감원, 법원, 국세청 ...
금전요구 키워드: 계좌이체, 안전계좌, 공탁금, 송금, 이체, 입금 ...
개인정보 키워드: 주민번호, OTP, 카드번호, 비밀번호, 계좌번호 ...
```

**장점**
- 추론 속도 < 10ms (온디바이스 실시간 처리 가능)
- 해석 가능 — 어떤 키워드에 걸렸는지 즉시 확인 가능
- Precision 97% — 오탐이 거의 없음

**한계**
- 키워드를 우회한 표현은 탐지 불가  
  예) *"안전한 곳으로 자산을 옮겨드릴게요"* → `공탁금`, `이체` 없으므로 미탐
- 신조어·변형 표현에 취약
- F1(macro) 85% — 카테고리 경계에서 오탐/미탐 발생

**역할**: 1차 필터로 31점 이상 감지 시 KoELECTRA 추론 트리거. 단독 배포 기준으로는 불충분.

---

### ② KoELECTRA 베이스 (파인튜닝 전)

**방식**: `monologg/koelectra-base-v3-discriminator` 사전학습 가중치 + 랜덤 초기화된 3-class 분류 헤드.  
보이스피싱 데이터로 학습하기 전 상태.

**결과 해석**
- Recall = 100% / Precision = 72% → **모든 샘플을 피싱으로 예측하는 편향**
- 랜덤 헤드의 초기 가중치가 우연히 sigmoid > 0.5를 유발해 전부 양성 출력
- F1(macro) = 27% — 카테고리별로는 일부 레이블을 전부 0, 일부를 전부 1로 예측하는 극단적 패턴

**의미**  
사전학습 언어모델이 한국어 문법과 의미를 이해하더라도, **보이스피싱 도메인 지식 없이는 랜덤 추측 수준**임을 보여줌. 파인튜닝의 필요성을 정량적으로 증명하는 기준선(baseline).

---

### ③ KoELECTRA 파인튜닝 (현재 모델)

**방식**: ② 베이스 모델을 `train_ml.json` (451개) 으로 파인튜닝.

```
손실함수 : BCEWithLogitsLoss (멀티라벨)
에폭     : 최대 10 (EarlyStopping patience=3)
학습률   : 3e-5 / 배치 16 / 최대 토큰 128
평가기준 : F1 (macro)
```

**결과**
- F1(bin) = 100% — 피싱/정상 이진 분류 완벽
- F1(macro) = 95% — 3개 카테고리 평균
- 카테고리별: 기관사칭 93% / 금전요구 95% / 개인정보 97%

**장점**
- 키워드 없는 우회 표현도 문맥으로 탐지
- 기관사칭 / 금전요구 / 개인정보를 동시에 독립 판단 → 복합 수법 대응

**한계 및 다음 단계**
- 현재 PyTorch 모델 (440MB) → TFLite int8 양자화 필요 (목표 20MB 이하)
- 테스트셋 57개 기준이므로 실제 배포 전 더 넓은 평가 권장
- TFLite 변환 후 추론 속도 80ms 이하 달성 여부 확인 필요
